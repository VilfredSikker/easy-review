Features

Everything you need, nothing you don't

‚¨°

AI-powered review

Run /er-review from Claude Code to get per-file risk ratings and hunk-level findings rendered inline. Press a to toggle the AI findings layer ‚Äî independent of comments and panel.

üí¨

Two comment types

Press c to leave GitHub comments (cyan) that sync with your PR. Press q to leave private questions (yellow) for AI review. Reply to threads with r.

1

Five diff modes

Branch diff, unstaged, staged, commit history, and merge conflicts. Switch with 1 2 3 4 5 ‚Äî see exactly what matters.

w

Live watch mode

Diffs refresh automatically as files change. Perfect next to Claude Code ‚Äî see changes as they're written.

‚á•

Multi-repo tabs

Open multiple repos or worktrees side-by-side. Navigate between them with [ ].

‚úì

Review tracking

Mark files as reviewed with Space, filter to unreviewed with u. Never lose your place.

‚óá

Syntax highlighting

Language-aware coloring via syntect. Diffs look like they do in your editor, not a wall of green and red.

‚Üï

Line-level navigation

Arrow keys navigate individual lines within hunks. Comment on exact lines, not just hunks. nN jumps between hunks.

f

Composable filters

Press f to filter by glob, status, or size (+*.rs,-*.lock,>50). Press F for built-in presets and filter history. Reviewed counter tracks both filtered and total progress.

PR

Smart PR integration

Full PR panel with CI checks, reviewer status, and labels. Auto-detects the right base branch and shows a hint when your diff target differs from the PR target.

p

Context panel

Press p to open the side panel with PR overview, AI summary, file detail, or symbol references. Press Tab to focus the panel and navigate its content.

‚áÖ

Two-way GitHub sync

Press G to pull review comments from GitHub. Press P to push your local comments back. Reply threads included. Powered by the gh CLI ‚Äî no API token needed.

5

Conflicts mode

Press 5 to enter Conflicts mode. Shows merge conflicts with resolved/unresolved indicators, staging support, and a merge status badge in the top bar.

‚öì

Comment relocation

A 3-pass algorithm keeps comments anchored to the right line as code changes underneath them. Comments dim with a warning when their target line disappears entirely.

g

Symbol references

Press g to run git grep for the word at cursor. Results appear in the context panel so you can see all usages without leaving the diff.

S

Live settings

Press S to open the settings overlay. Toggle feature flags, adjust display options, and set agent config. Changes apply live and persist to .er-config.toml.

Workflow

Built for AI-assisted development

1

Split your terminal

Claude Code on the left, er on the right. Ghostty, tmux, zellij ‚Äî whatever you use.

2

Enable watch mode

Press w and diffs update live as the AI writes code. No manual refreshing.

3

Get AI review

Run /er-review in Claude Code. Risk ratings, findings, and a review checklist appear inline in your diff view. Press a to toggle the AI findings layer ‚Äî independent of comments and panel.

4

Comment and iterate

Press c for GitHub comments or q for private questions on hunks and lines. Open the context panel with p for a PR overview alongside your diff. Re-run /er-review ‚Äî the AI reads your feedback and updates its analysis.

5

Review, stage, commit

Navigate files with jk, lines with ‚Üë‚Üì, hunks with nN. Stage what looks good with s, mark reviewed with Space.

6

Sync with GitHub

Press G to pull review comments from the PR into your diff view. Press P to push your local comments back to GitHub. The full review loop, without leaving the terminal.
