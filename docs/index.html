Comments

Leave comments
where they matter

Two comment types, one keyboard. GitHub comments sync to your PR. Private questions go to AI review. Both anchor to the exact line — and follow it through 3-pass relocation when code shifts underneath.

  • ✓ Two comment types — c for cyan GitHub comments that sync to your PR, q for yellow private questions answered by AI
  • ✓ 3-pass relocation — comments stay anchored as code shifts. Dim with a stale warning when their target line disappears entirely
  • ✓ Line-level navigation — arrow keys move between lines. Comment on exact lines, not just hunks. Tab to enter comment focus mode
Keys: c q r d
PR integration

Your PR, without
leaving the terminal

CI checks, reviewer status, labels, and GitHub comment sync — all inside er. Auto-detects your base branch from the PR target. Push to remote with Ctrl+P when you're done.

  • ✓ Smart PR integration — auto-detects base branch, shows CI status, reviewer decisions, and labels without opening a browser
  • ✓ Two-way GitHub sync — G pulls review comments from GitHub into your diff view. P pushes your local comments back. Powered by the gh CLI
  • ✓ Git push from TUI — stage, commit, and Ctrl+P push without leaving er

Philosophy

Lightweight by design —
configure everything,
bundle nothing

er is a 4MB binary. It doesn't bundle an AI, a test runner, or a linter. It's a bridge: your tools write JSON to .er/, er reads it and renders it inline. Claude Code writes review findings. You write comments. GitHub PR review state syncs both ways.

  • ✓ Live settings overlay — press S to toggle feature flags, adjust display options, and set agent config. Changes apply immediately and persist to .er-config.toml
  • ✓ Per-repo config — .er-config.toml in your repo root overrides global ~/.config/er/config.toml. Watched file globs for git-ignored paths like .work/
  • ✓ .er/ JSON bridge — all sidecar files live in .er/ (single .gitignore entry). Any tool can write to them — Claude Code, CI scripts, custom scripts. er picks up changes via mtime polling

Workflow

How it fits into your day

1

Split your terminal

Claude Code on the left, er on the right. Works with Ghostty, tmux, zellij — whatever you already use.

2

Watch mode is on by default

Diffs refresh as Claude Code writes. No manual refresh, no switching windows.

3

Run /er-review in Claude Code

Risk ratings, hunk-level findings, and a review checklist appear inline. Press a to toggle the AI layer independently of your comments and panel.

4

Comment and iterate

Press c for GitHub comments or q for private questions on any line. Open the context panel with p for a PR overview alongside your diff. Re-run /er-review — the AI reads your questions and updates its analysis.

5

Review, stage, commit, push

Navigate files with jk, lines with ↑↓, hunks with nN. Jump to next unreviewed with U. Stage with s, mark reviewed with Space, then Ctrl+P to push — without leaving the TUI.

6

Sync with GitHub

Press G to pull review comments from the PR into your diff view. Press P to push your local comments back to GitHub. The full review loop, without leaving the terminal.
