easy-review documentation

Review diffs at the speed of thought

easy-review (the er binary) is a git diff review tool built for developers who work with AI coding assistants. AI writes code faster than you can read it — er makes review fast, navigable, and live-updating, whether you prefer a keyboard-driven terminal UI or a full desktop application.

New here? Start with Installation, then walk through your first review in the Quick Start. If you just want the key map, jump to the Keybinding Reference.

Two front ends, one engine

er ships as two applications that share the same review engine, the same on-disk review data, and the same core concepts. Pick the one that fits how you work — or use both on the same repository and branch, since they read and write the same review files.

Terminal

The er TUI

A single static binary. Keyboard-driven, instant startup, runs anywhere a terminal does — perfect next to Claude Code in a split pane.
Desktop

The desktop app

A Tauri application with split diffs, an embedded terminal and browser, multi-model review “arena”, and a settings GUI.

What you can do with it

  • Read diffs in six modes — branch, unstaged, staged, commit history, merge conflicts, and hidden/ignored files.
  • Follow changes live — watch mode refreshes the diff as files are saved, staged, and committed.
  • Track your progress — mark files reviewed, filter to what is left, jump to the next unreviewed file.
  • Leave two kinds of notes — private questions for yourself or your AI, and GitHub comments that sync to a pull request.
  • Layer in AI analysis — per-file risk levels, inline findings, a review checklist, and specialized expert agents.
  • Work with pull requests — open a PR by number or URL, see CI and reviewer status, and sync comments two ways.

A note on your data

er is a viewer and orchestrator. It does not run AI models itself, and it does not transmit your code anywhere. There is no telemetry, no analytics, and no backend — it shells out to your local git (and optionally the gh CLI) and reads review artifacts that AI tools you run write to disk. You bring your own AI tools and credentials. See AI Review for how that loop works.
