Core Concepts

AI Hub Actions

Open the AI Hub — press a in the TUI or Cmd+A (macOS) / Ctrl+A (Windows/Linux) in the desktop app — to run any of these actions. Each writes sidecar files that er reads and renders inline. The Hub is the primary way to trigger AI work; no external tool or command is needed.

The standard loop

StepAI Hub / AppActionWhat happens
1 AI Hub TUI: Review work · Desktop: Run review Writes review sidecar files; findings appear inline
2 er v Cycle review panel; browse findings and inline comments
3 er q / c Leave questions and comments on lines
4 AI Hub (TUI only) Answer questions Reads your questions, responds, archives them.
Desktop: use the Notes panel instead — there is no Hub action for this.
5 er — Auto-refreshes; AI responses appear inline
6 Not a Hub action TUI: P key
Desktop: Comments card → Push to GitHub 		Validates freshness, posts findings to the GitHub PR as inline comments

Full catalog

All actions are available in both apps unless noted. Where TUI and Desktop differ, the difference is called out explicitly.

Run review

AppInvocation
TUIPress a → select Review work
DesktopPress Cmd+A / Ctrl+A → select Run review

Full review: risk levels, findings by severity, review order, checklist, and a markdown summary. Writes review.json, order.json, checklist.json, and summary.md.

Specialized / expert review

Eight expert lenses, each focused on a single dimension and writing only to experts/<id>.json. The TUI and Desktop offer the same experts but through different UI flows.

◆

TUI: Press a → Specialized review → a sub-menu lists all 8 experts; select one to run it.
Desktop: Press Cmd+A / Ctrl+A → Run reviewers… → a Choose reviewers multi-select appears (General + 8 experts + Professor); toggle with Space and confirm with Run N reviewer(s). You can run multiple in one go.

Triage

AppInvocation
TUIPress a → Triage branch
DesktopPress Cmd+A / Ctrl+A → Triage branch

Fast branch scan: first impression, routing verdict (ready for full review / needs work / trivial), priority files. Writes triage.json. Useful before committing to a full review.

Professor

AppInvocation
TUIPress a → Professor
DesktopPress Cmd+A / Ctrl+A → Professor (opens an optional focus-prompt modal before running)

A learning agent — it teaches rather than critiques. Surfaces concepts, patterns, and insights from the diff. Writes professor.json. See skills/PROFESSOR_PHILOSOPHY.md for the philosophy behind it.

Generate summary

◆

TUI only: Press a → Generate summary to regenerate summary.md as a standalone action.
Desktop: There is no standalone "Generate summary" action in the AI Actions palette. summary.md is produced automatically as part of Run review.

Answer questions

◆

TUI only: Press a → Answer questions. This action is disabled until you have added at least one question with the q key. It reads your questions, generates AI responses, and updates questions.json.
Desktop: There is no Hub action for answering questions. Questions are managed in the Notes panel — add questions with q and AI responses appear there.

Validate / re-anchor findings

AppInvocation
TUIPress a → Validate review
DesktopPress Cmd+A / Ctrl+A → Validate / re-anchor

Re-checks existing findings against the current diff and re-anchors line references that have moved. Useful after making changes in response to review feedback.

Generate guided tour

◆

Desktop only: Press Cmd+A / Ctrl+A → Generate tour (or Regenerate tour if one already exists). A "Re-run guide" button also appears in the Guide tab when the tour is stale. Writes tour.json.
TUI: The TUI can navigate an existing tour in Guide / Tour mode, but it cannot generate one. Generate the tour in the desktop app first.

Arena (multi-reviewer consensus)

◆

Desktop only — and not in the AI Actions palette. The Arena lives in the Review panel Arena card: use Run as Arena or Promote to Arena there.
TUI: Arena is not available in the TUI.

Runs multiple reviewer agents in adversarial rounds: experts propose findings, fresh same-lens skeptics try to refute them, an arbiter rules kept / dropped / merged. Higher-confidence results than a single-pass review. See Arena in the desktop features guide.

Publish to GitHub (not a Hub action)

Publishing review comments to a GitHub PR is not an AI Hub action — it is a separate operation in both apps:

  • TUI: Press P to push local comments to the PR; press G to pull comments from GitHub.
  • Desktop: Open the Comments card → Push to GitHub, then choose Push as review or push individual comments. Comments are pulled automatically when you open a PR tab.

Comprehension quiz

Quizzes have no in-app generation. Quiz content comes from the external TechProfessor integration or from imported sidecar files. The TUI Quiz mode (press 8) only renders an existing quiz.json — it cannot create one.

To import quiz sidecars written by an external tool: press I in the TUI or use Import local review files in the desktop app.

Review philosophy

Every reviewer shares one severity model and a "what not to flag" list, documented in skills/REVIEW_PHILOSOPHY.md and skills/REVIEW_RULES.md:

  • P0 (high) = must fix · P1 (medium) = should fix · P2 (low) = nice to fix.
  • Never flag naming, formatting, style, or file moves.

The professor follows a separate skills/PROFESSOR_PHILOSOPHY.md — it teaches rather than critiques.

Multi-reviewer runs

You can run the general review, any experts, and the professor concurrently. er merges each fresh sidecar (matching the current diff hash) into the in-memory review, and every finding carries an agent pill — General, Security, Professor, and so on — so you can tell at a glance which agent raised it. The desktop app makes this a first-class flow: use Run reviewers… in the AI Actions palette to select and launch multiple reviewers at once, or use the review arena for adversarial multi-model consensus.

Testing without running a review

To see the overlays without invoking an AI tool, generate fixtures with a matching diff hash:

cd your-repo
bash /path/to/easy-review/scripts/generate-test-fixtures.sh
