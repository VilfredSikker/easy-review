Core Concepts

Claude Code Skills

The AI half of er's feedback loop is a set of Claude Code skills — slash commands like /er-review. Each generates sidecar files that er reads and renders. This page is the full catalog and how the skills fit together.

Setup

Skills are auto-discovered from .claude/commands/ in the repository root, so anyone who clones the repo and uses Claude Code gets them with no installation. The source of truth is the skills/ directory; the files in .claude/commands/ are symlinks to each skills/<name>/SKILL.md.

◆

The skills document their output as .er/<file> relative to the repo. With default managed storage, run reviews through er's own agent commands, or set ER_REPO_LOCAL=1 so skill output lands where er reads it. See Review Storage.

Comprehension quiz (optional)

A workflow for checking that you actually understood a change before approving it.

SkillWhat it does
/er-quizGenerate comprehension questions about the P0/P1 changes → quiz.json
/er-quiz-reviewEvaluate your answers and write teaching feedback → quiz-feedback.json

In the terminal, press 8 for quiz mode, answer the questions, then run /er-quiz-review. Feedback appears inline after the next refresh.

Review philosophy

Every skill shares one severity model and a “what not to flag” list, documented in skills/REVIEW_PHILOSOPHY.md and skills/REVIEW_RULES.md:

  • P0 (high) = must fix · P1 (medium) = should fix · P2 (low) = nice to fix.
  • Never flag naming, formatting, style, or file moves.

The professor follows a separate skills/PROFESSOR_PHILOSOPHY.md — it teaches rather than critiques.

Multi-reviewer runs

You can run the general review, any experts, and the professor concurrently. er merges each fresh sidecar (matching the current diff hash) into the in-memory review, and every finding carries an agent pill — General, Security, Professor, and so on — so you can tell at a glance which agent raised it. The desktop app makes this a first-class flow with its multi-model review arena.

Testing without running skills

To see the overlays without invoking an AI tool, generate fixtures with a matching diff hash:

cd your-repo
bash /path/to/easy-review/scripts/generate-test-fixtures.sh
