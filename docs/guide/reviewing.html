Core Concepts

Reviewing & Navigation

The day-to-day of a review: moving through files and hunks, tracking what you have covered, narrowing the list to what matters, and jumping out to your editor when you need to. The concepts on this page apply to both apps; the keys shown are the terminal defaults. The desktop app offers the same actions through its UI and its own shortcuts — see Desktop Features.

Navigating the diff

The three panels — file list, diff, context panel — are driven entirely from the keyboard.

KeysAction
k / jNext / previous file
n / NNext / previous hunk
↓ / ↑Next / previous line within hunks
h / lScroll the diff left / right
d / uScroll down / up (also Ctrl+d / Ctrl+u)

A sticky file-path header keeps the current file's name pinned at the top of the diff as you scroll, so you always know where you are. Line-level navigation with the arrow keys is what lets you attach comments to an exact line rather than a whole hunk.

Marking files reviewed

Review is stateful. As you finish each file, record it:

KeyAction
SpaceToggle the current file as reviewed
!Filter the list to unreviewed files only
UJump to the next unreviewed file

The status bar shows a running reviewed / total count (respecting any active filter). Reviewed state is persisted per repository and branch, so you can close er mid-review and resume later.

◆

Auto-unmark: if watch mode detects that a file's diff actually changed, that file's reviewed mark clears automatically. New changes can never hide behind a stale checkmark.

Searching

Press / to search files by name. Type to filter the list incrementally; Esc clears the search. This is a quick way to jump to a known file in a large changeset.

Filtering

Filtering is more powerful than search: it is a composable expression over glob, status, size, and risk. Press f to type a filter. Combine clauses with commas; prefix with + to include and - to exclude. 

# only Rust files
+*.rs

# Rust files, but not lock files, and only files with > 50 changed lines
+*.rs,-*.lock,>50
ClauseMeaning
+glob / -globInclude / exclude files matching a glob pattern
>N / <NFiles with more / fewer than N changed lines
status & risk rulesFilter by file status or AI risk level

Press F for built-in presets and your filter history. You can also pre-filter from the command line: er --filter '*.rs'. Clear an active filter with Esc (innermost first — it clears search before filter).

Sorting

  • By recency — m toggles modification-time sorting in any mode, so the files your AI just touched rise to the top.
  • By risk — when an AI review has run, files carry risk indicators (high / medium / low) in the list. Running a review from the AI Hub also produces order.json, a suggested review order with groupings. See AI Review.

Editor & clipboard integration

KeyAction
eOpen the current file in $EDITOR
yOpen the copy hub — copy the file, its path, the current hunk, or the current line
RManually refresh the diff

Copying a hunk is handy for pasting a specific change into a chat with your AI assistant or a teammate. Opening in your editor jumps you straight to the file you are reviewing.

Tabs & multiple repositories

You can open more than one repository or git worktree at once and treat them as tabs: 

er ~/projects/api ~/projects/frontend
KeyAction
] / [Next / previous tab
xClose the current tab
oOpen hub — switch repository, worktree, or recent project

Each tab keeps its own review state, and watch notifications surface across tabs so you know when another repo or worktree has new changes. The desktop app extends this with persistent, drag-to-reorder tabs — see Desktop Features.

Settings hub

Press , to open the settings hub and adjust display and feature options live. Changes apply immediately and can be persisted to the global config from inside the hub. The full list of options is in Configuration.
