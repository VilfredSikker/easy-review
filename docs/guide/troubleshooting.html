Troubleshooting & FAQ

Answers to the questions that come up most often, and fixes for the things that occasionally go sideways.

Troubleshooting

er shows the wrong base branch

The base is auto-detected: upstream tracking → main → master → develop → dev. If it picks the wrong one, set your branch's upstream so tracking wins: git branch --set-upstream-to=origin/main. When a PR is detected with a different target, er shows a hint rather than switching automatically.

AI findings do not appear

  • Check where the artifacts went. By default er reads from managed storage, not the repo's .er/. External tools (such as TechProfessor) may write to .er/ in the repo instead. To bridge the gap: run reviews via the AI Hub (a in the TUI, Cmd+A / Ctrl+A in the desktop), or launch er with ER_REPO_LOCAL=1 to read the repo directory directly, or import the sidecar files with I in the TUI (Import local review files in the desktop). See Review Storage.
  • Check freshness. If the diff changed after the review ran, the data is dimmed as stale. Re-run the relevant action from the AI Hub.
  • Cycle the view. In the terminal, press v to make sure you are in an AI view mode.
  • Try fixtures. Run scripts/generate-test-fixtures.sh in your repo to confirm the overlay renders at all.

GitHub features are missing or failing

GitHub integration requires the gh CLI, authenticated with gh auth login. If G / P do nothing or PR data is absent, run gh auth status to confirm you are logged in and that gh is on your PATH.

The diff is not updating

Watch mode should refresh on saves, staging, and commits. Confirm the ● WATCH indicator is on (toggle with w), and press r to force a manual refresh. Watch uses a ~500ms debounce, so a burst of changes lands as one update.

A huge file is collapsed and I can't see it

Lock files, generated code, and files over ~500 lines are auto-compacted — shown as a single collapsed entry to keep things fast. Press Enter on it to expand on demand. See large-diff performance.

er exits immediately or renders nothing

The terminal UI needs a real TTY. Running it through a pipe or a non-interactive context will not work — launch it in an actual terminal (a tmux pane is fine).

The target/ directory is enormous (building from source)

Building the whole workspace compiles the Tauri desktop app into the shared target/, which can balloon. Use the scoped scripts: ./scripts/er-tui.sh for TUI work and ./scripts/tauri-dev.sh for desktop, and run ./scripts/cargo-gc.sh to reclaim space.

Getting a debug log

Run ER_DEBUG=1 er. It writes /tmp/er_debug.log (overwritten on each git diff) and shows the memory budget in the status bar.

FAQ

Does er run AI or send my code anywhere?

No. er does not run AI models and has no backend, telemetry, or analytics. It shells out to your local git (and optionally gh) and reads artifacts that AI tools you run write to disk. You bring your own AI tool and credentials, and you are the customer of those providers.

Which AI tools work with it?

Any tool you can drive from the command line. The included skills target Claude Code, and the AI Hub config ships presets for Claude, Codex, and Cursor. See AI Hub configuration.

Do I have to use AI at all?

No. Every non-AI feature — diff modes, navigation, reviewed tracking, comments, GitHub sync, filtering — works on its own. AI review is an optional layer on top.

Can I use the terminal and desktop apps together?

Yes. They share one engine and the same on-disk review data per repo, branch, and view, so you can switch between them mid-review.

Where is my review data stored, and is it in git?

By default it is in managed application data outside your repository, so it never touches git status. Set ER_REPO_LOCAL=1 to use a repo .er/ directory instead (add it to .gitignore if you do). Full paths are in Review Storage.

What does the desktop app add over the terminal?

Split and word diffs, an embedded terminal and browser, the multi-model review arena, a settings GUI, persistent drag-to-reorder tabs, and reference “find usages”. The terminal wins on speed, zero dependencies, and running anywhere a terminal does. See the Desktop Overview.

Is the desktop app released?

Not yet — published releases are the er terminal binary only. Build the desktop app from source with ./scripts/tauri-dev.sh or ./scripts/tauri-build.sh.

How do I report a bug or request a feature?

Open an issue on the GitHub repository.
