Core Concepts

Review Storage

Reviewed marks, comments, questions, AI artifacts, and session state are stored as files on disk. By default they live in managed application data — not in your repository — and are shared between the terminal and desktop apps per repo, branch, and view.

Why managed storage?

Keeping review sidecars out of your working tree means they never clutter git status, never need a .gitignore entry, and never risk being committed by accident. Both front ends resolve the same path for a given tab, so a review you start in the desktop app continues seamlessly in the terminal.

Where the root is

PlatformStorage root
Linux~/.local/share/easy-review/
macOS~/Library/Application Support/easy-review/
Windows%APPDATA%\easy-review\

Layout for local diff modes

Branch (1), unstaged (2), staged (3), and history (4) each get their own view bucket directory under the repo and branch: 

<root>/repos/<repo-slug>/branches/<branch-slug>/view-buckets/<bucket>/

where <bucket> is branch, unstaged, staged, or history. For example, on Linux for the easy-review repo on main, branch diff:

~/.local/share/easy-review/repos/easy-review/branches/main/view-buckets/branch/

Layout for PR diff mode

Pull request reviews are keyed by owner/repo and PR number, so a local clone and a remote PR tab share one location:

<root>/repos/<owner-repo-slug>/prs/pr-<N>/

The <repo-slug> comes from the origin remote name (falling back to the repo folder name). Slashes in branch names become -.

The sidecar files

The filenames are identical in managed storage and in repo-local mode:

Each JSON sidecar stores a SHA-256 diff_hash of the diff it was generated against. When the diff changes, the matching AI data is dimmed with a stale warning. Writes are atomic (write to a temp file, then rename), so a crash mid-write never corrupts your data.

Repo-local mode

Set ER_REPO_LOCAL=1 to read and write a .er/ directory inside the repository instead of managed storage. This is useful for debugging, or for external tools (such as the TechProfessor integration) that write sidecar files into .er/ relative to the repo. 

ER_REPO_LOCAL=1 er
▲

In repo-local mode, add .er/ to your .gitignore so review artifacts are not committed.

◆

With default managed storage, run reviews through er's built-in AI Hub (TUI: a / Desktop: Cmd+A). Use ER_REPO_LOCAL=1 only when an external tool writes directly into .er/ and you want er to read from the same location.

Migration from older versions

Earlier versions scattered sidecar dotfiles in the repo root, then consolidated them into a repo .er/ directory. Legacy repo .er/ and ~/.cache/er/ data is migrated once, automatically, the first time you open a repo with a current build.

Tests & overrides

Set ER_STORAGE_ROOT=/tmp/... to redirect managed storage — primarily used by the test suite.

Cleaning up

From the terminal you can remove artifacts for the current file (z) or all of them (Z). The repository also ships helper scripts:

scripts/er-cleanup-reviews.sh     # remove review.json, checklist, summary, …
scripts/er-cleanup-questions.sh   # remove questions.json
