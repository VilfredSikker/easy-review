Core Concepts

Configuration

er is configured with TOML — there are no required CLI flags. Settings are deep-merged: a global file provides shared defaults, and a per-repo file overrides individual fields. The desktop app exposes most of these through a settings GUI.

Where config lives

LocationScopePriority
.er-config.toml (repo root)Per-repoHighest
~/.config/er/config.tomlGlobalMiddle
built-inDefaultsLowest

Merging is deep and per-field: a repo file only needs to specify what differs from your global config. Everything else inherits from global, then from the built-in defaults. A complete annotated template ships as config.toml.example in the repository.

Live editing terminal

Press , inside er to open the settings hub. Changes apply immediately so you can see the effect, and s persists them to your global config.

Settings GUI desktop

The desktop app has a dedicated settings page (command palette → Open settings) covering the same options organized into scopes — general options shared with the terminal (theme, display, AI Hub providers and models, concurrency), desktop-only options, and embedded-terminal options. Changes apply live and are written back to the same TOML files, so the two apps always agree.

[features] — which modes and features are on

Feature toggles. All default to true.

[features]
view_branch    = true   # Branch diff
view_unstaged  = true   # Unstaged changes
view_staged    = true   # Staged changes
view_history   = true   # Commit history
view_conflicts = true   # Merge conflicts (tab appears during a merge)
view_hidden    = true   # Hidden / watched files (tab appears when [watched] paths exist)
view_tour      = true   # Guided Tour walkthrough (tab appears when a tour.json exists)
arena          = true   # Multi-reviewer arena (desktop)

Disabling a view mode removes its tab and renumbers the rest — there are no gaps in the number keys.

[display] — rendering

[display]
theme        = "graphite"  # graphite (default) | slate | midnight | ember | paper | daylight | contrast-dark | contrast-light
line_numbers = true
wrap_lines   = false       # wrap long lines instead of scrolling horizontally
split_diff   = false       # side-by-side diff view
tab_width    = 4           # spaces per tab (1–16)
auto_context_threshold = 100  # auto-expand context for small files, tiered by diff size (0 disables)

The theme drives both the terminal and desktop UIs: four dark themes (Graphite, the default, plus Slate, Midnight, Ember), two light (Paper, Daylight), and two AAA high-contrast variants (Contrast Dark, Contrast Light). Each maps to a matching syntax-highlighting theme. Older theme names (ocean-depth, moonlight, daybreak, high-contrast, …) still resolve via aliases.

[hints] — bottom-bar key hints terminal

[hints]
navigation = true   # j/k, n/N, Space, /
comments   = true   # q, c, r, x
staging    = true   # s, commit
verbose    = false  # show all keybindings

[agent] and [ai_hub] — AI providers

[agent] sets the single command used to run AI review actions. {prompt} is replaced with the generated prompt. 

[agent]
command = "claude"
args    = ["--print", "--output-format", "stream-json", "-p", "{prompt}"]

The optional [ai_hub] block defines presets — multiple providers and their models — so you can switch at runtime without editing config. Provider args are the shared base; each model's args are appended after them. 

[ai_hub]
default_provider = "claude"
default_model    = "sonnet-5"
default_effort   = "medium"  # reasoning effort passed to providers that support it
max_concurrent_reviews = 3  # queue further reviews beyond this many at once (1–16)

[ai_hub.reviewer_models]
triage = "haiku-4.5"          # pin specific reviewer kinds to specific models

[ai_hub.providers.claude]
label   = "Claude"
command = "claude"
args    = ["--print", "-p", "{prompt}"]

[[ai_hub.providers.claude.models]]
id    = "sonnet-5"
label = "Sonnet 5"
args  = ["--model", "claude-sonnet-5"]
# optional metadata: description, cost_per_1k_in, cost_per_1k_out, avg_latency_ms
  • If [ai_hub] is absent, er falls back to the single [agent] config.
  • Selection is session-local; the presets themselves come from TOML.
  • On load, er merges any missing built-in catalog models into your config in memory — it does not rewrite your file.
  • Providers in the example config include Claude, Codex, and Cursor.
  • The selected provider/model applies to AI Hub actions: review, triage, experts, professor, questions, and summary.

[summary] — diff summaries

[summary]
# command = "claude"                        # defaults to agent.command
# args    = ["--print", "-p", "{diff}"]     # {diff} is replaced with the raw diff
push_to_pr = false                       # auto-push the summary to the GitHub PR body

[watched] — monitor git-ignored files

Glob patterns for ignored paths to watch — for example, an AI assistant's sync folder. Watched files appear in the file tree alongside tracked changes; toggle the section with W. 

[watched]
paths     = [".work/**/*", ".er/**/*"]  # patterns matched against the repo root
diff_mode = "content"                  # "content" (show file) or "snapshot" (diff vs saved baseline)
▲

Watched files should be in .gitignore. er runs a safety check and warns if a watched pattern is not actually ignored.

[commands] — hub shell actions

Optional shell commands for hub actions, run via sh -c. Leave one unset to hide that action. Placeholders: {base}, {branch}, {repo}, {output}. 

[commands]
# summary   = "claude --print -p 'Summarize this diff: {base}...{branch}'"
# test      = "cargo test"
# lint      = "cargo clippy"
# typecheck = "cargo check"
# security  = "cargo audit"

In a mono-repo, an optional [packages] section defines the same commands per package — each key (e.g. [packages.api]) takes its own label, test, lint, typecheck, and security commands, and the Verify hub groups them by package.

Worked example

Global (~/.config/er/config.toml) holds preferences shared across repos:

[features]
view_conflicts = false

[display]
tab_width  = 2
wrap_lines = true

[watched]
paths = [".work/**/*", ".er/**/*"]

Per-repo (.er-config.toml) overrides only what differs:

[display]
tab_width = 4

[watched]
paths     = [".work/**/*", ".er/**/*", "logs/**/*.log"]
diff_mode = "snapshot"

Result: view_conflicts and wrap_lines come from global; tab_width = 4 and the extra watched path come from the repo file.
