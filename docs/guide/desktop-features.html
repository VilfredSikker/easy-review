Desktop App

Features in Depth

A tour of the capabilities that are unique to the desktop app. For the shared review model — diff modes, comments, AI sidecars — see Core Concepts.

▲

The desktop app is evolving quickly. Treat specific shortcuts and panel names as current-as-of-writing rather than a frozen contract.

Diff rendering: split and word diff

Beyond the unified view, the desktop app offers a split (side-by-side) diff — old on the left, new on the right — toggleable from the keyboard or command palette. Within changed lines it computes an intra-line word diff, so a renamed variable or a tweaked literal is highlighted at the token level rather than marking the whole line changed. Syntax highlighting runs client-side via Shiki in a Web Worker, cached per file.

The review arena

The arena runs multiple AI reviewers over the same diff and reconciles their findings — think of it as a panel of reviewers rather than a single pass.

  • Configure a run — pick reviewers (models and/or specialized agents like Security, Performance, Testing), choose a scope (full branch, unstaged, staged, or selected files), set the number of rounds, and pick an arbiter model.
  • Cost preview — an estimate based on each model's per-1k-token pricing, with an approval gate before anything runs.
  • Watch progress — a minimizable panel shows the current round and phase, with a cancel button.
  • Read the results — three layouts: a bracket view of how verdicts flowed, a matrix of reviewers against findings, and a funnel from proposed to final.
  • History — past arena runs on the current branch are listed with their reviewer and finding counts.

Concurrency is bounded: spawns go through a queue with a configurable cap (default 3, adjustable in settings), and a process-wide semaphore keeps simultaneous runs from multiplying agent subprocesses.

AI Hub: providers, models, reviewers

The AI action palette (Cmd+A) is the desktop's front door to AI. From it you can run a review, launch the arena, choose a provider (Claude, Codex, Cursor), pick a specific model, select which reviewer kinds to run, and reach export, the browser, and settings. Running reviews surface as background tasks with cancellation; results land in the Review tab, filterable by severity or by agent. The provider and model catalog comes from your AI Hub configuration.

Inline threads & findings

Comments and questions render as inline threads on the diff, with a reply composer. Two desktop conveniences stand out:

  • Ask AI — prompt an agent to validate or expand on a thread directly from the comment.
  • Promote to GitHub — turn a private question into a GitHub PR comment when you decide it is worth raising.

AI findings render as inline cards with a severity badge and an agent pill (Security, Performance, Professor, …), their own reply thread, and a control to promote them into GitHub comments. Resolved and stale states are tracked just as in the terminal.

Embedded browser & page annotations

The desktop app can show a real embedded browser beside your diff — handy for reading docs, a design system, or an API while you review. It loads real http://localhost URLs so logins and sessions work like a normal browser, with an iframe proxy fallback. Its annotation mode lets you click elements on the page to drop pins, optionally capturing a screenshot crop and DOM context, then write a markdown note. Annotations can link back to specific lines in the diff, and findings in code can link out to page sections.

Embedded terminal drawer

A resizable terminal drawer runs a full shell (xterm.js + PTY) per tab, themed to match the app. Run tests, git commands, or anything else without leaving the review. It includes a generous scrollback buffer and a one-click “checkout this branch” helper.

Code references & “find usages”

A search bar (Cmd+F) prefills from selected text or a hovered identifier. Hovering an identifier highlights every occurrence in the diff, and a “find usages” popover lists references with surrounding line context (calls, assignments, and so on). A ruler along the scroll gutter marks where matches are.

Triage & professor

The Review tab can show a triage card with a routing verdict (general review, expert review, arena, professor, or skip), a confidence level, rationale, and priority files — each with a one-click button to run the recommended next step. A professor focus flow lets you pair an expert with a custom focus prompt and scope for a deeper, teaching-oriented pass.

Export a review

The export view renders a live markdown preview of your findings, comments, questions, and annotations, with toggles to include or exclude each type and an “unresolved only” filter. Copy it to the clipboard or save it to export.md — useful for a review summary in a PR description or a writeup.

Tabs, projects, and PRs

  • Persistent tabs — open repos, branches, and PRs as tabs that survive restarts; drag to reorder.
  • Lazy restoration — inactive tabs are lightweight stubs until selected, with background warming for fast switching.
  • Project / branch picker — switch repositories and branches from the sidebar.
  • Remote PR review — paste a PR URL (Cmd+Shift+O) and review against remote refs without a checkout.
  • Worktree picker — jump between git worktrees.

Settings GUI

The settings page exposes configuration as a UI, organized into tabs:

TabCovers
GeneralTheme, default AI provider and model, agent effort and concurrency cap. Shared with the terminal where applicable.
ProjectsPer-project triage timing, file-size limits, and review ignore patterns.
TerminalTUI display and key-hint options.

The same themes drive both the desktop and terminal UIs; changing one restyles the app live.
