Getting Started

Installation

The terminal app (er) is a single binary you can install in seconds. The desktop app is currently built from source. Both need git; everything else is optional.

Requirements

gitEverythingRequired. er shells out to git for all diff operations.

Quick install — macOS & Linux

The install script downloads a pre-built binary into ~/.local/bin/:

curl -fsSL https://raw.githubusercontent.com/VilfredSikker/easy-review/main/install.sh | bash

Install to a different directory, or pin a version:

# custom install directory
curl -fsSL https://raw.githubusercontent.com/VilfredSikker/easy-review/main/install.sh | bash -s -- --dir /usr/local/bin

# a specific released version
curl -fsSL https://raw.githubusercontent.com/VilfredSikker/easy-review/main/install.sh | bash -s -- --version v0.3.0
◆

Make sure your install directory is on your PATH. For ~/.local/bin, add export PATH="$HOME/.local/bin:$PATH" to your shell profile if it is not already there.

From source with Cargo

git clone https://github.com/VilfredSikker/easy-review.git
cd easy-review
cargo install --path crates/er-tui

Or install directly from the repository without cloning:

cargo install --git https://github.com/VilfredSikker/easy-review er-tui

This produces a single er binary with no runtime dependencies beyond git.

Verify the install

er --help
cd ~/some/git/repo
er

Install from source

Clone the repo and run the desktop app in development, or build a release bundle for your machine:

Run in development

git clone https://github.com/VilfredSikker/easy-review.git
cd easy-review
./scripts/tauri-dev.sh

This launches the Tauri dev shell with the Svelte frontend and hot reload.

Build a release bundle

./scripts/tauri-build.sh

On macOS this produces an Easy Review.app under the desktop build target (set ER_SKIP_DMG=0 to also produce a .dmg). Building requires cargo-tauri and the platform WebView dependencies (WebKit/GTK on Linux).

✦

The terminal and desktop apps read and write the same review data per repository and branch, so you can install both and switch between them freely. See Review Storage for where that data lives.

Updating

  • Script install: re-run the install command to fetch the latest release.
  • Cargo install: re-run cargo install --git … er-tui (add --force if needed).
  • From a clone: git pull then rebuild.

Uninstalling

Remove config, managed review storage, legacy cache, and installed apps from either surface:

Terminal

er uninstall              # interactive confirm
er uninstall --yes        # no prompt
er uninstall --dry-run    # list paths only
er uninstall --keep-data  # leave review storage
er uninstall --keep-apps  # leave binaries / .app

Or re-run the install script with --uninstall: 

curl -fsSL https://raw.githubusercontent.com/VilfredSikker/easy-review/main/install.sh | bash -s -- --uninstall

Desktop

Open Settings → General → Uninstall (also linked from the welcome screen). Confirm by typing uninstall; the app quits after removal. Paths in use by the running process are deleted after it exits.

What gets removed by default:

  • ~/.config/er/ (and the platform config dir) — config, projects, tabs, caches
  • Managed review storage under …/easy-review/ (see Review Storage)
  • Legacy ~/.cache/er/
  • The er binary (common install locations) and Easy Review.app when found

MCP server (er-mcp)

Optional local MCP server for Claude Code, Cursor, Codex, and OpenCode — PR triage queues and AI sidecar upload into shared managed storage. Full setup (one-line claude mcp add / codex mcp add, plus mcp.json for Cursor and opencode.json for OpenCode): MCP Server Setup. 

cargo install --path crates/er-mcp
