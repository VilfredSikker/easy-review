Terminal UI

Keybinding Reference

The complete key map for the er terminal UI, grouped by task. Bottom-bar hints show a subset; this is the full list. Hint groups are configurable — see Configuration. Press ? inside er for the built-in help hub, which always reflects the running version.

Diff modes

Number keys 1–9 switch between the visible mode tabs. Numbering is dynamic: tabs are numbered in the order they appear, and modes that do not apply are hidden without leaving gaps. On a typical local branch that means 1 Branch, 2 Unstaged, 3 Staged, 4 History — with extra tabs appearing when available: PR Diff (when the branch has a known PR), Guided Tour (when a tour.json exists), Conflicts (during a merge), and Hidden / watched files (when [watched] paths are configured). See Diff Modes.

KeyAction
1–9Switch to the Nth visible mode tab
mToggle sort by recency (modification time)
RRefresh the diff
wToggle watch mode
WToggle the watched-files section

Comments, questions & notes

Lowercase keys start a draft anchored to the current line (or selection). While composing, Ctrl+t cycles the draft type — question → note → GitHub comment — and Tab pauses the draft (press Tab again in normal mode to resume it). Uppercase keys toggle layer visibility, not creation. See Comments & Questions.

KeyAction
qStart a private question on the current line (yellow)
cStart a GitHub comment on the current line (cyan) — in Staged mode, c commits instead
Ctrl+tWhile composing: cycle draft type (question → note → comment)
QToggle visibility of the questions/notes layer
CToggle visibility of the GitHub-comments layer
AToggle visibility of the AI-findings layer
XHide / show resolved items
J / KFocus the previous / next inline item
rReply to the focused comment, question, or finding
eEdit the focused comment (your own, top-level) — otherwise opens the file in $EDITOR
xDelete the focused comment (with confirmation)

Hubs

Most one-off actions live in small modal hubs rather than on dedicated keys. Each hub lists its own options with single-key shortcuts.

KeyHub
aAI Hub — Review work, Triage branch, Specialized review, Professor, Answer questions, summaries, cleanup (see AI Hub Actions)
gGit hub — push to remote, stage files, refresh, pull / push GitHub comments, comment on or approve the PR
vVerify hub — run tests, linter, type check, and other configured [commands]
yCopy hub — copy the file, path, hunk, or line to the clipboard
oOpen hub — switch repository, worktree, or recent project
,Settings hub — live-edit configuration (see Configuration)
?Help hub — searchable list of every key and action

◆

Some keys are context-dependent: c comments everywhere except Staged mode, where it commits; x deletes a focused comment but closes the tab when nothing is focused; e edits a focused comment of your own but otherwise opens the file in your editor. The bottom bar and the ? help hub always reflect what is available right now.
