Getting Started

MCP server setup

er-mcp is a local stdio MCP server. Ask Claude Code, Cursor, or Codex for priority PRs, production-only diff size, blocked/stale filters, and to prepare/upload AI review sidecars into the same managed storage Desktop/TUI use.

Prerequisites

  • git and gh on PATH (gh auth login)
  • Optional: Easy Review projects at ~/.config/er/projects.json (Desktop writes this) so you can omit repo=
  • A built er-mcp binary (next section)

Install (recommended: npx)

No Rust toolchain needed. The easy-review-mcp npm package downloads the matching er-mcp binary from GitHub Releases on first run. 

# Smoke-test
npx -y easy-review-mcp

# Cursor — ~/.cursor/mcp.json (or .cursor/mcp.json)
{
  "mcpServers": {
    "easy-review": {
      "command": "npx",
      "args": ["-y", "easy-review-mcp"]
    }
  }
}

# Claude Code
claude mcp add --scope user easy-review -- npx -y easy-review-mcp

# Codex
codex mcp add easy-review -- npx -y easy-review-mcp

Requires Node 18+. Supported hosts: macOS arm64/x64, Linux x64. Override with ER_MCP_PATH=/path/to/er-mcp if you already built from source.

Install from source (optional)

git clone https://github.com/VilfredSikker/easy-review.git
cd easy-review
cargo install --path crates/er-mcp
# → ~/.cargo/bin/er-mcp  (ensure ~/.cargo/bin is on PATH)

which er-mcp
er-mcp --help

From a clone without installing globally:

cargo build -p er-mcp --release
# binary: target/release/er-mcp
◆

When using a source-built binary, set $ER_MCP to its absolute path (e.g. $HOME/.cargo/bin/er-mcp) in the client commands below. Prefer the npx form above when you can.

Fast add (source binary)

Claude Code

# User scope — available in every project
claude mcp add --scope user easy-review -- "$ER_MCP"

# Project scope — writes .mcp.json for the team (commit it)
claude mcp add --scope project easy-review -- "$ER_MCP"

claude mcp list
claude mcp get easy-review

Flags go before the server name; everything after -- is the stdio command. There is no --command flag.

Codex

codex mcp add easy-review -- "$ER_MCP"
codex mcp list
codex mcp get easy-review

Config is stored in ~/.codex/config.toml (or project .codex/config.toml for trusted projects).

Cursor

Cursor uses mcp.json (no first-party cursor mcp add equivalent for custom stdio servers). Fastest path: paste the JSON below, or add via Cursor Settings → Tools & MCP → New MCP Server.

# Global (all workspaces)
mkdir -p ~/.cursor
# then edit ~/.cursor/mcp.json — see structure below

# Or project-local (share with the team)
mkdir -p .cursor
# then edit .cursor/mcp.json

mcp.json structure

Same shape for Cursor global (~/.cursor/mcp.json), Cursor project (.cursor/mcp.json), and Claude project scope (.mcp.json at the repo root).

Minimal (recommended — npx)

{
  "mcpServers": {
    "easy-review": {
      "command": "npx",
      "args": ["-y", "easy-review-mcp"]
    }
  }
}

Source-built binary

{
  "mcpServers": {
    "easy-review": {
      "command": "/absolute/path/to/er-mcp"
    }
  }
}

With args / env (optional)

{
  "mcpServers": {
    "easy-review": {
      "command": "/absolute/path/to/er-mcp",
      "args": [],
      "env": {
        "ER_STORAGE_ROOT": "/custom/easy-review-data"
      }
    }
  }
}

Where each client reads it

ClientConfig locationNotes
Cursor ~/.cursor/mcp.json (global)
.cursor/mcp.json (project)		 Project wins on name clash. Restart or toggle the server in Settings if it does not appear.
Claude Code claude mcp add → user/local in ~/.claude.json; project → .mcp.json Prefer claude mcp add over hand-editing. In-session: /mcp.
Codex ~/.codex/config.toml ([mcp_servers.easy-review]) Prefer codex mcp add. CLI / IDE / ChatGPT desktop share this host config.

Codex TOML equivalent

# ~/.codex/config.toml
[mcp_servers.easy-review]
command = "npx"
args = ["-y", "easy-review-mcp"]

Install the agent skill

MCP exposes the tools; the er-review skill teaches agents to run prepare → author → upload when you say “ER review”. Same install path as github/gh-stack: 

# Global (user) — prompts for which agents to install into
npx skills add VilfredSikker/easy-review -s er-review -g

# Project scope (omit -g)
npx skills add VilfredSikker/easy-review -s er-review

# Non-interactive examples
npx skills add VilfredSikker/easy-review -s er-review -g -a cursor -y
npx skills add VilfredSikker/easy-review -s er-review --all

Skill source: skills/er-review/SKILL.md in the repo.

Verify

  1. Restart the client (or reload MCP tools).
  2. Ask: “List Easy Review projects” or “Top 5 priority PRs”.
  3. You should see tools like priority_prs, prepare_review, upload_artifacts.
  4. After npx skills add …, try: “ER review” on a PR number.

Troubleshooting

  • Server not listed — try npx -y easy-review-mcp in a terminal; or check the absolute path / restart client.
  • Tools fail on GitHub — run gh auth status; MCP shells out to gh like Desktop.
  • No projects — pass repo=owner/name, or open the repo in Easy Review Desktop once so projects.json exists.
  • Claude “unknown option --command” — use claude mcp add easy-review -- "$ER_MCP" (options before name, command after --).
