Terminal UI

The er Terminal UI

er is a single, dependency-free binary that renders a fast, keyboard-driven review UI in your terminal. It starts in under a second and is designed to sit in a split pane next to an AI coding assistant. This page covers what is specific to the terminal app; the shared concepts live under Core Concepts.

Launching

Run er from inside any git repository. Common invocations:

er                                        # review the current branch diff
er --pr 42                                # open a GitHub PR by number
er https://github.com/owner/repo/pull/42  # open a PR by URL
er ~/projects/api ~/projects/frontend     # open multiple repos as tabs
er --filter '*.rs'                        # pre-filter the file list
Flag / argumentEffect
--pr <n>Open pull request n in the current repo
a PR URLOpen that pull request (any repo)
one or more pathsOpen each repository / worktree as a tab
--filter <expr>Apply a filter at startup
--helpShow all options
▲

er renders via crossterm/ratatui and needs a real terminal (such as a tmux pane), not a headless pipe.

The layout

The screen has a top bar, three columns, and a bottom bar.

RegionShows
Top barThe diff modes (with the active one highlighted), any detected PR with its CI summary, comment and finding counts, and the watch indicator.
Left — file listEvery changed file with a status dot, AI risk indicator, and comment/finding badges. In history mode this becomes the commit list.
Center — diffThe unified diff with syntax highlighting, inline comments and findings, and a sticky file-path header.
Right — context panelFile details, the AI summary, or the PR overview depending on context.
Bottom barKey hints (configurable) and comment input when you are typing a note.

Working beside an AI assistant

The intended workflow is a split terminal: Claude Code (or your tool of choice) on one side, er on the other, in tmux, Ghostty, or any terminal that splits panes. As the assistant edits, stages, and commits, er's watch mode keeps the diff live. Open the AI Hub with a and run a review — findings appear in er automatically.

AI view modes

Press v (forward) or V (backward) to cycle how much AI context is shown — from a plain diff, through inline findings, to a side panel with the summary and risk breakdown, to a dedicated AI review layout. This is the terminal's way of dialing AI density up and down. See AI Review.

Syntax highlighting

The terminal uses syntect for highlighting, with a content-hash cache so unchanged lines are never re-highlighted between frames. The active theme (set in configuration) selects a matching highlight palette.

Debug mode

Run with ER_DEBUG=1 to write a debug log and show the memory budget in the status bar:

ER_DEBUG=1 er   # writes /tmp/er_debug.log (overwritten each git diff)
